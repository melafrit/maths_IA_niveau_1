Contenu de la séance (cours magistral + démonstrations)

Notions abordées :
  • Transition : de la régression (prédire un nombre) à la classification (prédire une catégorie)
  • Classification binaire : y ∈ {0, 1}, seuil 45 k€ sur le dataset salaire
  • Rappels de probabilités : P(A) ∈ [0,1], estimation empirique, espérance, variance
  • Probabilité conditionnelle : P(A|B), application P(sal>45k | exp>5) ≈ 85%
  • Classification linéaire : fonction de décision g(x) = wᵀx + b, règle du signe
  • Frontière de décision : l'hyperplan wᵀx + b = 0 (une droite en 2D)
  • Le perceptron (Rosenblatt, 1957) : ŷ = signe(wᵀx + b), algorithme d'apprentissage par correction d'erreurs
  • Règle de mise à jour : w ← w + α(yᵢ − ŷᵢ)xᵢ
  • Limites du perceptron : le problème XOR (non linéairement séparable) → teaser Jour 4
🎯 Fil rouge — Application sur le dataset salaire
• On crée la variable binaire salaire_eleve (seuil 45 k€) : 15 vs 15, dataset équilibré.
• On calcule les probabilités conditionnelles : l'expérience est très prédictive.
• On entraîne un perceptron from scratch (100 époques, accuracy ~87-90%).
• On montre que le perceptron échoue sur XOR : introduction au Jour 4.
Compétences visées :
  • Passer d'un problème de régression à un problème de classification
  • Calculer des probabilités conditionnelles sur un dataset
  • Implémenter un perceptron et comprendre la règle de mise à jour
  • Visualiser et interpréter une frontière de décision
